# 안드로이드 USB 디버깅 설정 가이드

WebADB 서비스를 사용하기 위해서는 안드로이드 기기에서 USB 디버깅을 활성화해야 합니다.

## 📱 USB 디버깅 활성화 방법

### 1단계: 개발자 옵션 활성화

대부분의 안드로이드 기기에서 개발자 옵션은 기본적으로 숨겨져 있습니다.

1. **설정** 앱을 엽니다
2. **휴대전화 정보** 또는 **디바이스 정보** 메뉴로 이동합니다
3. **빌드 번호**를 7번 연속으로 탭합니다
4. "개발자가 되었습니다!" 메시지가 표시됩니다

#### 제조사별 경로

**Samsung (삼성)**
```
설정 → 휴대전화 정보 → 소프트웨어 정보 → 빌드 번호 (7번 탭)
```

**LG**
```
설정 → 휴대전화 정보 → 소프트웨어 정보 → 빌드 번호 (7번 탭)
```

**Google Pixel**
```
설정 → 휴대전화 정보 → 빌드 번호 (7번 탭)
```

**Xiaomi (샤오미)**
```
설정 → 내 기기 → 전체 사양 → MIUI 버전 (7번 탭)
```

**Huawei (화웨이)**
```
설정 → 휴대전화 정보 → 빌드 번호 (7번 탭)
```

### 2단계: USB 디버깅 활성화

1. **설정** 앱으로 돌아갑니다
2. **개발자 옵션** 메뉴를 찾습니다
   - 일반적으로 설정의 하단 또는 시스템 메뉴 안에 있습니다
3. **USB 디버깅** 옵션을 찾아 활성화합니다
4. 확인 대화상자에서 **확인**을 탭합니다

### 3단계: PC와 연결

1. USB 케이블을 사용하여 안드로이드 기기를 PC에 연결합니다
2. 기기에서 **USB 디버깅 허용** 대화상자가 표시됩니다
3. **이 컴퓨터에서 항상 허용** 옵션을 체크합니다 (선택사항)
4. **확인**을 탭합니다

## ⚙️ 추가 설정 (선택사항)

### USB 설정 모드 변경

일부 기기에서는 USB 연결 모드를 변경해야 할 수 있습니다:

1. USB 케이블 연결 후 알림 패널을 내립니다
2. **USB 옵션** 또는 **USB 연결** 알림을 탭합니다
3. **파일 전송** 또는 **PTP** 모드를 선택합니다

### 보안 설정

일부 기기에서는 추가 보안 설정이 필요할 수 있습니다:

**개발자 옵션 → USB 디버깅 (보안 설정)**
- 일부 제조사에서는 "USB 디버깅 (보안 설정)" 옵션을 제공합니다
- 이 옵션도 함께 활성화하세요

## 🔍 문제 해결

### USB 디버깅이 활성화되지 않음

**원인 1: 권한 부족**
- 일부 회사 기기나 관리되는 기기에서는 USB 디버깅이 차단될 수 있습니다
- IT 관리자에게 문의하세요

**원인 2: Knox/MIUI 보안**
- Samsung Knox나 Xiaomi MIUI의 보안 정책으로 제한될 수 있습니다
- 보안 설정에서 추가 권한을 허용하세요

### USB 디버깅 허용 대화상자가 나타나지 않음

**해결 방법 1: USB 디버깅 권한 초기화**
```
개발자 옵션 → USB 디버깅 권한 취소
```
그 다음 USB 케이블을 다시 연결하세요

**해결 방법 2: 다른 USB 포트 사용**
- PC의 다른 USB 포트에 연결해보세요
- USB 3.0 포트보다 USB 2.0 포트가 더 안정적일 수 있습니다

**해결 방법 3: USB 케이블 확인**
- 데이터 전송을 지원하는 USB 케이블인지 확인하세요
- 일부 충전 전용 케이블은 데이터 전송을 지원하지 않습니다

### 기기가 Chrome에서 인식되지 않음

**Windows 드라이버 설치**
1. 제조사 웹사이트에서 USB 드라이버를 다운로드합니다
2. 드라이버를 설치합니다
3. 기기를 다시 연결합니다

**주요 제조사 드라이버 다운로드 링크:**
- Samsung: https://developer.samsung.com/mobile/android-usb-driver.html
- LG: LG PC Suite 설치
- Google Pixel: 별도 드라이버 불필요
- Xiaomi: Mi PC Suite 설치

## 🔒 보안 주의사항

### USB 디버깅의 위험성

USB 디버깅을 활성화하면:
- ✅ 개발 및 테스트가 용이해집니다
- ⚠️ 보안 위험이 증가할 수 있습니다

### 보안 모범 사례

1. **신뢰할 수 있는 컴퓨터에만 연결**
   - 공공 USB 충전 포트 사용 시 주의하세요

2. **이 컴퓨터에서 항상 허용 옵션 신중히 사용**
   - 자주 사용하는 본인의 PC에만 체크하세요

3. **사용 후 비활성화**
   - 테스트 완료 후 USB 디버깅을 비활성화하는 것이 안전합니다

4. **USB 디버깅 권한 주기적 초기화**
   ```
   개발자 옵션 → USB 디버깅 권한 취소
   ```

## 📋 체크리스트

WebADB 사용 전 다음 사항을 확인하세요:

- [ ] 개발자 옵션이 활성화되어 있음
- [ ] USB 디버깅이 활성화되어 있음
- [ ] USB 케이블이 데이터 전송을 지원함
- [ ] 기기가 PC에 연결되어 있음
- [ ] USB 디버깅 허용 대화상자에서 확인을 탭함
- [ ] Chrome 브라우저를 사용 중 (버전 61 이상)

## 🎯 빠른 설정 요약

```
1. 설정 → 휴대전화 정보 → 빌드 번호 (7번 탭)
2. 설정 → 개발자 옵션 → USB 디버깅 활성화
3. USB 케이블로 PC 연결
4. 기기에서 USB 디버깅 허용
5. Chrome에서 WebADB 서비스 접속
```

## 🆘 여전히 문제가 있나요?

위의 방법으로 해결되지 않으면:

1. 기기를 재시작해보세요
2. PC를 재시작해보세요
3. 다른 USB 케이블을 시도해보세요
4. 제조사별 커뮤니티 포럼에서 정보를 찾아보세요
